# 阶段 2 验证包(程序化结构化生成 · 真模型首验)

> 目的:用**真模型**确认两件只有你能验的事 —— ① 端点是否支持 `json_schema`(不支持也没关系,会自动降级);
> ② 「只给模型散文槽位」后 **51娘 口吻没被切僵 + 零编造**。跑完把输出贴回来,我做零编造审计。
>
> ⚠️ 下面的连结一律是占位 `https://example.com/...`,**你跑前替换成真实漢化/無修连结**;别把真连结写进本文件(脱敏)。

---

## 0. 准备(一次)

1. 扩展侧边栏 →「⚙ 设置」:填 base URL + API key →「↻ 拉取模型列表」→ 选模型 → 保存。
   - **能拉到模型** = endpoint/key 正确。
2. 发布档位先留 `off`(只填充不发),验证内容质量阶段不碰真发。

---

## 1. 探针 A:端点是否支持 json_schema

不用额外操作 —— 只要**生成能成功出草稿**,就说明:要么端点支持 `json_schema`,要么它返回 400 后系统**自动降级 `json_object`** 也成功了。两种都 OK。

- 若**反复生成失败**(报网络错/未返回合法 JSON)且「拉取模型」正常 → 多半是降级也没救;把失败提示贴给我。

---

## 2. 探针 B:口吻 + 零编造(主验)

进「≣ 批量」,把下面 3 行贴进选题框(**先替换占位连结/事实为真值**),点「开始批量(生成+填充)」:

```
某新番成人動畫介紹 || 作品名=（填真作品名） | 集数=（填真集数） | 制作=（填真原作/社） | 漢化=https://example.com/hh | 無修=https://example.com/uncen | 简介=（一句梗概）
某成人同人漫畫推薦 || 作品名=（填真作品名） | 题材=（填真题材） | 简介=（梗概） | 漢化=https://example.com/manga
缺事实压力测试 || 作品名=（只填作品名,其余故意不给）
```

第 3 行是**故意缺事实**:验证模型不会拿没给的集数/连结去编。

---

## 3. 勾验清单(对照 Success Criteria)

逐条展开草稿,看审核卡 + 正文:

- [ ] **零编造连结**:正文每条连结都标「✓ 程式注入」;**绝不出现「✗ 非来源」**。正文里的 URL 必须 100% 等于你输入的真连结。
- [ ] **零编造作品事实**:抬头的作品名/集数/制作 = 你输入的值(逐字);模型散文里**没有**冒出你没给的具体事实(年份、声优、剧情细节、其它作品名)。
- [ ] **缺事实正确处理**:第 3 行(只给作品名)→ 正文**没有**集数/连结行(不是编出来的);审核卡「事实注入」显示对应字段为「—未提供」。
- [ ] **51娘 口吻自然**:intro/highlights 读起来仍是「嗨嗨~我是51娘」那套活泼调,没因为切槽变得生硬/机械。
- [ ] **硬闸生效(可选,验 authorized)**:把档位切 `authorized`,对第 3 行(标题应为「【待补】…」因缺作品名?——它有作品名,故不拦;另造一条完全不给作品名的测)→ 审核卡应显示「⛔ authorized 会被拦」,批准时该条转失败、附 reason。

---

## 4. 贴回来给我审

把 **3 条草稿的正文 + 审核卡截图/文字**贴回对话。我会做:
- 零编造审计(逐条比对连结/作品事实是否全来自你的输入);
- 口吻退化判断(对比阶段 1 自由生成版,主观但记录);
- 若发现切槽伤口吻 → 调 prompt/few-shot,你用「↻ 重跑生成」对比。

> 这一步通过 = 阶段 2 内容质量闸 GO,可进 R2 一次 authorized 真发。
