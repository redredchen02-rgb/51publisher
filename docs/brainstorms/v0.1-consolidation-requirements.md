### 2. Backend 內部結構收斂

將 flat 的 `packages/backend/src/` 重整為分層結構：

```

├── index.ts              # 極簡入口：讀 env、呼叫 app.build()、listen()

│
├── config/
│   ├── env.ts            # 環境變數載入與驗證（原 env-check.ts）
│   └── schemas.ts        # TypeBox schema 定義（原 schemas.ts）
│
├── middleware/
│   ├── auth.ts           # JWT 認證（原 auth-middleware.ts）
│   └── error-handler.ts  # 全域錯誤處理（合併 error-response.ts 邏輯）
│
├── routes/
│   ├── auth.ts           # 認證相關路由

│   ├── config.ts         # 動態配置路由
│   ├── published-posts.ts# 已發布文章路由
│   ├── pending.ts        # 待審題庫路由（原 scraper/pending-routes.ts）
│   ├── prompt.ts         # Prompt 路由（原 scraper/prompt-routes.ts）

├── scrapers/             # 抓取子系統（隸屬 services 語意，但獨立目錄）
```

### 3. index.ts 瘦身

當前 `index.ts` 306 行，承擔了以下四項職責：
- Server 建構（Fastify 實例化 + 插件註冊）→ **移至 `app.ts`**
- 路由註冊 → **移至 `app.ts`**
- Scraper 初始化與 cron 啟動 → **移至 `app.ts`**
- `start()` 啟動函數 → **保留在 `index.ts`，但只呼叫 app.build().listen()**

重整後 `index.ts` 應只剩 15~25 行。

### 4. app.ts 新增

建立 `app.ts`，導出 `buildApp()` 工廠函數。所有路由註冊、中間件掛載、scraper/scheduler 初始化在 app.ts 中完成。

## 實施策略

**Wave 1：版本號（5 分鐘）**
- 編輯 5 個檔案（root/3 packages/VERSION/CHANGELOG）
- 只需字串替換，無依賴順序

**Wave 2：backend 目錄重整（45 分鐘）**
- 建立新目錄結構（config/middleware/routes/services/stores/utils）
- 建立 `app.ts`，從 `index.ts` 抽離 server 建構邏輯
- 搬移所有源檔到對應目錄
- 更新所有 `import` 路徑
- 使用 `git mv` 保留 git 歷史

**Wave 3：驗證**
- `lsp_diagnostics` 檢查變更檔案
- `pnpm compile` backend
- `pnpm -r compile` 全專案
- `pnpm --filter publisher-backend test`
