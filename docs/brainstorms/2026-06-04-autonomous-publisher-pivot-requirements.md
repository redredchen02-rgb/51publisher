# 自主发布器转向 — 借 Webwright 之力(Hybrid C)

## Problem Frame

51publisher 当前是「只填充、绝不自动提交」的 Chrome 扩展,人必须逐条点发布。
帖子全部发往**自家站点**(可随时撤下/改/删,无封号风险),因此「逐条人工点发布」
是产能瓶颈,而非安全必需。研究微软 **Webwright**(SWE 风格、模型自主端到端完成网页任务、
Playwright、工作区即状态、任务=可重跑脚本)后,决定**移除零提交铁律**,让产品从
「填充助手」进化为「自家站点的自主发布器」。

Webwright 的**代码/agent 无法直接搬入**(Python+Playwright 的外部自主 agent vs. 本产品的
WXT 浏览器扩展;且 Webwright 会自动提交,正是旧铁律所禁)。本次是**移植理念**,不是导入代码。

## 方向决策(已拍板)

形态走 **C 混合**:抽出可移植 recipe(站点知识)→ Webwright 式自主循环驱动 →
人工**批量审核**(而非逐条点)→ 信任阶梯护栏。扩展在过渡期保留,终态可向无头(B)收敛。

```
选题队列 → AI 生成 → 脱敏 → 填充 → [批量审核闸门] → 自动发布(先草稿) → 落盘轨迹 → 一键转正式
```

## Requirements

**移除铁律 / 重新定义闸门**
- R1. 移除「零提交」硬约束:系统在自家站点可执行**自动提交/发布**动作。
- R2. 人工闸门从「逐条点发布」改为「**批量审核**」(v1 默认档位):agent 批量填充+截图 N 条草稿,
  人审一次后整批放行。批量大小可配置(抽样审核、全自动零审核为可选档位,非 v1 默认)。
- R3. 旧的「零提交测试」由新的**安全档位测试**取代:断言系统只在被显式授权的站点+模式下提交。

**Webwright 理念移植(三件)**
- R4. **可移植 recipe**:把 51acgs 站点知识(字段映射、Quill 写入流程、脱敏白名单)抽成
  执行层无关的 recipe,扩展与无头 agent 都能消费。
- R5. **填充/发布轨迹存档(工作区即状态)**:每次任务落盘可审计轨迹(填了什么字段、前后截图、
  绿/黄/红结果、最终发布 URL),可回放、可回滚依据。
- R6. **可重跑漂移探针**:把字段映射升级成能对 fixture/真后台**重放**的自检脚本,
  **主动**发现后台改版导致的选择器漂移(替代当前"被动发现")。

**信任阶梯护栏(刹车,非铁律)**
- R7. **急停开关**:任意时刻一键停止当前批次的后续发布。
- R8. **dry-run 预演**:跑完整流程但不真正提交,产出"将要发布什么"的报告。
- R9. **先草稿后转正**:默认发布为草稿/不公开态;提供"一键批量转正式"。发错时读者不可见。
  (依赖自家 CMS 是否支持草稿态 — 见 Outstanding Questions)
- R10. **可回滚**:基于 R5 轨迹,提供"撤下/改回上一版"的批量操作或操作指引。

## Success Criteria

- 在自家测试站点,能**无人值守批量发布** ≥ N 条(N 待定),全程零手动点击提交。
- 一次后台改版后,**漂移探针主动报警**,而不是等真实发帖失败才发现(R6 见效)。
- 任一错误批次能在 5 分钟内**全部撤下或转回草稿**(R7+R9+R10 见效)。
- 产能相对"逐条人工发布"显著提升(批量审核闸门不成为新瓶颈)。

## Scope Boundaries

- **不**支持第三方平台的自动发布(封号/反爬/难撤回风险);本次仅限**自家站点**。
- **不**追求 Webwright 的通用 web-agent 能力(任意网站任意任务);只做"发帖到自家 CMS"这一窄任务。
- **不**在 v1 直接做无头 B 终态;C 是去 B 的路,过渡期扩展保留。
- 封面/媒体上传是否纳入自动化,本次不扩张(沿用现状,除非 R4 recipe 顺带覆盖)。

## Key Decisions

- **移除零提交铁律**:因帖子全发自家站、可随时撤下,逐条人工发布是瓶颈而非安全必需。
- **形态选 C 而非 B/A**:C 复用已啃下的站点硬知识、人工闸门仍在(只是批量化)、可平滑过渡到无头。
- **护栏取代铁律**:用"先草稿后转正 + 急停 + dry-run + 轨迹回滚"换取自动化吞吐,而非全瞎自动发。
- **Webwright 定位**:作引擎/理念来源(可重跑脚本、工作区即状态、SWE 式自愈),非直接导入代码。
- **v1 闸门档位 = 批量审核**:抽样/全自动作为可选档位,非默认。
- **目标量级 = 几条~几十条/天(中低量)**:C 形态足够,**不急于**收敛到无头 B;架构留好升 B 的口即可。

## Dependencies / Assumptions

- 假设自家 CMS(51acgs 等)的会话/登录可被自动化复用(cookie/session)。
- 假设站点改版频率使"漂移探针"有维护价值(过去"被动发现"是痛点 → 大概率成立)。
- 现有资产可复用:字段映射、`lib/fillers.ts` 写入流程、DOMPurify 脱敏白名单、e2e fixture + 真 Quill。
