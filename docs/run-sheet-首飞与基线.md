## Step 0 · 准备(一次,~2 分钟)

1. 扩展已是今天的构建(`.output/chrome-mv3/`)。`chrome://extensions` →「加载已解压」选该目录(或已加载就跳过)。
2. 侧边栏「⚙ 设置」:base URL + API key →「↻ 拉取模型列表」→ 选模型 → 保存。**能拉到模型 = 配置正确。**
3. 档位先留 **`off`**(只填充不发)。基线阶段**完全不碰真发**。
4. 后台 `https://dx-999-adm.ympxbys.xyz/` 发帖页登录、打开「添加」表单(登录你自己来)。

**判定准则(人工)**:

- **质量**:`直发`=逐字属实、口吻对、可直接发;`小改`=个别措辞调一下;`不合格`=要重写 / 含编造 / 缺关键事实。
- **编造**:正文每条连结应标「✓ 程式注入」、URL 100% = 你输入;作品名/集数/制作 = 你输入(逐字);散文里**不该冒出你没给的事实**(年份/声优/剧情/别的作品名)。第 3 条(缺事实)应**没有**集数/连结行,而非编一个。
- **degrade**:填充结果表里分类/标签若黄(无匹配),把模型吐的那个**自由文字**记下来(如「成人動畫」「校園/日常」)——这是 G4 词表的真底料。

**手工对照基线(一次)**:挑 1–2 个同题,你**纯手写**一篇,记总耗时:**\_\_** 分钟/篇。(用来判「仪器到底省不省时」。)

## Step 2 · 算一下 → GATE go/no-go

- 合格率 =(直发 + 小改)/ 总数 = \_\_\_\_%　**(目标 ≥ 80%,你拍板)**
- 单条净耗时(找事实 + 审改)中位数 = \_**\_ 分钟　vs 手工 \_\_** 分钟　**(目标 ≤ 手工,否则即便能发也没替代手工)**
- 幻觉发生率 = 出现编造的条数 / 总数 = \_\_\_\_%
- degrade 命中的分类/标签清单:********\_\_\_\_********

**判定**:合格率达标 + 净耗时 ≤ 手工 →「每天能用」成立;否则记下卡在哪(找事实太久?口吻?幻觉?degrade?)。

## Step 3 · 可选 · 一次 authorized 首飞(G2 · 不可逆 · 你亲手)

> 拿 Step 1 里一条 `直发` 草稿(或单写一条安全测试),**status 先设「隐藏」**——真走 `POST /save` 验证管路,但读者看不到、可删。
