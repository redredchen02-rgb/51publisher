# 安装与使用指南

51publisher 是一个 Chrome 扩展:用大模型按「**51娘**」口吻生成帖子草稿,人工审核后填进后台、再由你决定是否发布。

> 🛡️ **防幻觉核心(程序化结构化生成)**:模型**只负责写口吻散文**;作品名、集数、连结等事实由**系统从你提供的事实里原样填入正文,模型碰不到**——从流程上让它没机会编造连结或作品事实。缺的事实不会出现在正文里,审核区会标出缺口。

---

## 一、环境要求

- **Chromium 内核浏览器**(Chrome / Edge 等)——正文写入依赖主世界脚本,**Firefox 不支持**。
- **Node ≥ 20** 与 **pnpm**(仅"自己构建加载"时需要)。

---

## 二、安装(加载未打包扩展)

```bash
pnpm install
pnpm build          # 产出 .output/chrome-mv3/
```

然后在浏览器:

1. 打开 `chrome://extensions`,右上角开启「**开发者模式**」。
2. 点「**加载已解压的扩展程序**」,选择项目里的 `.output/chrome-mv3/` 目录。
3. 工具栏点扩展图标 → 打开 **side panel(侧边栏)**。

> 之后若代码有更新:重新 `pnpm build`,再到 `chrome://extensions` 点该扩展的**刷新 ↻**。

---

## 三、首次配置(⚙ 设置)

打开侧边栏 → 右上角「**⚙ 设置**」,依次填:

| 项                | 怎么填                                                                                                                                                                            |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **endpoint**      | 大模型地址,**填到 base URL 即可**(如 `https://api.openai.com/v1`),会自动补 `/chat/completions` 和 `/models`。必须 `https://`。                                                    |
| **API key**       | 你的 key。⚠️ 明文存于本地浏览器(`chrome.storage.local`),并随请求发往你配置的 endpoint——**只配可信地址,建议用权限受限的专用 key**。                                                |
| **模型**          | 填好上面两项后,点「**↻ 拉取模型列表**」→ 在下拉里选一个;拉不到也可手填模型名。                                                                                                    |
| **Prompt 模板**   | 已内置「51娘 + 只写口吻散文槽位」契约,通常无需改。模型只回 intro/highlights 等散文,**不回整篇正文、不写连结**——正文由系统组装。占位符:`{{topic}}` / `{{facts}}` / `{{fewshot}}`。 |
| **Few-shot 范例** | 已内置脱敏范例(示口吻);可改,但**别写真实漢化/無修连结**(会随请求外发)。                                                                                                           |
| **字段映射**      | 默认值来自现场勘查,后台改版时才需按指南更新。                                                                                                                                     |

点「**保存**」。能成功拉到模型,说明 endpoint / key 都对了。

## 四、使用(批量源接地流程)

侧边栏进入 **≣ 批量** 视图。

### 1. 输入选题 + 事实

每行一条。**核心:把你掌握的事实喂给它,AI 只负责润色,不会编造。**

```
住在拔作島上的我該如何是好成人動畫介紹 || 作品名=住在拔作島上的我該如何是好 | 集数=2期 | 漢化=https://… | 無修=https://…
精靈寶可夢成人同人推薦 || 作品名=精靈寶可夢 | 题材=同人本 | 简介=莉莉艾/瑪俐/奇樹
某新番看点(只写选题也行,缺的事实 AI 会标【待补】,不会瞎编)
```

- 分隔:`选题 || 字段=值 | 字段=值`。
- 支持字段:`作品名` / `集数`(话数)/ `制作`(原作)/ `漢化` / `無修` / `题材`(标签)/ `简介`。
- 别名也认:`name` / `ep` / `tags` / `desc` 等。

### 2. 生成 + 审核

点「**开始批量(生成+填充)**」。**模型只写口吻散文,作品名/集数/连结由系统从你的事实原样填入正文(模型碰不到,无法编造)。** 展开每条草稿,看审核卡:

- **事实注入状态**:每个字段标 ✓已注入 / —未提供,一眼看到缺口(缺的字段不会出现在正文里)。
- **连结**:正文每条连结标「✓ 程式注入(不可编造)」;若出现「✗ 非来源」即异常,务必核实。
- **发布前硬闸**:显示该条若 authorized 发布会否被拦(标题缺作品名 / 正文残留【待补】 / 无来源连结 → 拦,需补全或编辑后再发)。
- **填充三态表**:绿=已填 / 黄=跳过 / 红=降级。

可直接在卡片里编辑草稿字段。

### 3. 选发布档位(在 ⚙ 设置里切)

| 档位         | 行为                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| `off`        | **只填充,不发布**(最安全,默认)。需人工去后台手动点发布。                    |
| `dry-run`    | 预演:走完整流程但**不真发**,产出 DryRunReport 看"将发布什么"。              |
| `authorized` | **真实发布**(仅当目标标签页停在授权站点)。批准时需打字 `publish` 二次确认。 |

> ⚠️ **authorized 真发后无法自动撤回**。强烈建议先 `dry-run` 预演,首次真发把后台状态设「隐藏」,核对无误再显示。

### 4. 批准

切到 `dry-run` 或 `authorized` → 批准前会自动做**选择器漂移自检**(后台改版会被挡下)→ 通过后批准。authorized 需输入 `publish` 手势。

### 5. 调 prompt 后重跑对比

改了设置里的 Prompt / Few-shot,想看效果?在待审状态点「**↻ 重跑生成(改 prompt 后对比)**」——用同批题目只生成不发,肉眼对比新旧效果(已发过的题目也能重跑)。

### 6. 历史

切到「**历史**」标签看已发布轨迹(可校验链完整性、点开帖子核对)。

## 五、安全与边界

- **不自动提交/发布**(除非你显式切到 authorized 并打字确认);第三方平台一律只填充。
- **源接地防幻觉**:AI 只用你给的事实 + 套口吻;缺的标【待补】、连结只用你给的 URL。审核卡会红标任何编造的连结。
- API key 明文存本地、随请求外发——**只配可信 endpoint**。
- 正文 HTML(最不可信输入)写入编辑器前按白名单消毒(防 XSS)。

## 六、常见问题

| 现象                               | 多半原因 / 解法                                                                                                                            |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 「拉取模型列表」或生成报**网络错** | endpoint 写错;或该 LLM 域名不放行跨域(CORS)。后者需把该域名加进 `wxt.config.ts` 的 `host_permissions` 后重新 build。                       |
| 生成偶发失败但「拉取模型」正常     | 你的端点可能不支持 `json_schema` 响应格式;系统会自动降级为 `json_object` 重试,通常无需处理。仍失败多为模型不稳,重试或 Prompt「恢复默认」。 |
| 生成报「未返回合法 JSON」          | 模型不稳或 prompt 被改坏;重试或在设置点 Prompt 模板「恢复默认」。                                                                          |
| 草稿连结被红标 ✗                   | AI 用了不在你输入里的 URL=疑似编造,**别发**,改用你的真连结或留【待补】。                                                                   |
| 正文显示「需手动」                 | 极端情况 Quill 不可用,走兜底写入;建议点「复制正文」手动粘贴。                                                                              |
| 后台改版后填充错位                 | 多为选择器漂移;轻则改「⚙ 设置 → 字段映射」,重则需改代码(见 `docs/field-mapping-guide.md`)。                                                |
